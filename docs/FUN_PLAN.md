# 有料化計畫(2026-06-12,user:「挺失敗的,先去找怎麼做一個有料的遊戲」)

## 誠實診斷:現在為什麼無聊

研究結論(MapleStory 設計分析 / game feel 文獻 / ARPG combat design)對照現狀:

1. **玩家的手沒事做**。ARPG 核心樂趣 = 「控制空間」:怪會圍你、你得走位/跳/拉怪。
   現在自動攻擊 + 怪傷害無感(5-6 點)→ 站著就贏 → 按不按「避」都一樣 → 無聊。
   楓之谷好玩不是因為自動,是因為**跳跳打打 + 怪會擠死你**。
2. **打擊感只做半套**。有白閃/傷害字,但**沒 HitStop、沒音效、沒擊退、沒死亡演出**。
   文獻:音效佔 juice 的 50%;Vlambeer:screenshake+hitstop+音效三件套缺一不可。
3. **內容密度趨近零**。1 張圖、1.5 種怪(換色)、1 把武器、掉落只有貨幣。
   ARPG 之魂是**撿裝備的驚喜**(詞綴隨機、稀有度顏色),現在完全沒有。
4. **沒有目標梯**。分鐘級(任務/掉寶)、場次級(boss/每日)、長期(build)全空。

## 三層修法(優先序)

### A. 手感層 — 地基,先做
- **跳躍**:輕功二段跳 + 跳劈(空中攻擊)。楓之谷的靈魂。平台 2-3 層。
- **HitStop**:60ms × 0.05 timescale 全 sub-system sync(老專案驗證過的配方)。
- **音效**:Sonniss/Freesound 免費 SFX——刀風、肉擊、怪嚎、撿錢叮、升級鐘。
- **擊退**:打怪退 40px、怪打你退 60px;怪死亡 = 墨散化開 + 戰利品實體噴出(走過去撿)。
- **連擊計數**:連殺墨字 combo(3 連斬!五連斬!),斷了歸零。

### B. 威脅層 — 讓手必須動
- 怪傷害 ↑ 到單次 15-25% 血;藥水(攜帶上限)= 資源決策。
- **怪型三件套**(每隻一句話讀懂的招):
  - 妖狐:撲擊(現有)
  - 衝撞型(山魈):直線衝鋒,跳開或吃大傷
  - 遠程型(墨鴉):拋物線吐墨,逼走位
- **圍毆**:aggro 連鎖(打一隻引兩隻),2-3 隻同時上 → 走位/跳/避全部有意義。
- 原則(howtomakeanrpg):怪移速 < 玩家;特殊快招(衝鋒)給冷卻;攻擊前搖一致可讀。

### C. 內容/目標層
- **裝備掉落**:武器/防具 白綠藍紫金 5 稀有度 + 隨機詞綴,掉在地上發光待撿。
- **3 張圖鏈**:青丘霧林(Lv1-10)→ 赤目嶺(10-20)→ 九尾祠(boss)。各自怪表+主視覺。
- **主線任務 v1**:殺 N 隻交付 → 獎勵 + 解鎖下一圖(分鐘級目標)。
- **每日試煉** + boss 計時重打(場次級目標)。

## 鐵則
- 自動攻擊保留(user 拍板防連點器),但**自動只是底色**:跳劈/技能/走位/撿裝備全是手動的爽。
- 每做一項 → 實測 → 才算數。手感項先全做完再動內容層。
